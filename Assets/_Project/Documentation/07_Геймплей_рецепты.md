# 07. Геймплей-рецепты

Практические «как сделать» — для типовых задач jam-игры.

## Добавить игровой счёт

1. В геймплей-коде вызови `GameManager.Instance.AddScore(amount)`.
2. `GameManager` сам пошлёт в `OnScoreChanged`, а `HUDScreen` уже подписан и обновит лейбл.

## Завершить игру

- `GameManager.Instance.GameOver()` — выставит state в `GameOver`, обновит best score.
- `GameManager.Instance.Victory()` — аналогично, но state = `Victory`.

`EndGameScreen` подписан на `OnGameStateChanged` и сам покажется поверх HUD.

## Загрузить сцену

- `SceneLoader.Instance.LoadMainMenu()`
- `SceneLoader.Instance.LoadGame()`
- `SceneLoader.Instance.Load("custom_scene_name")`
- `SceneLoader.Instance.RestartCurrentScene()`

На `_onLoadStart` можно показать `LoadingScreen`, на `_onLoadComplete` — спрятать.

## Добавить менеджер-синглтон

1. Класс наследуется от `Singleton<MyManager>` (см. [`Singleton.cs`](../Scripts/Core/Singleton.cs)).
2. Компонент нужно положить на `_Managers.prefab` — только тогда он переживёт смену сцен.
3. Обращаться как `MyManager.Instance.Foo()`. Перед первым обращением из `Awake` других объектов — проверяй `MyManager.HasInstance`.

## Object Pooling

[`ObjectPool.cs`](../Scripts/Core/ObjectPool.cs) — простой стек-пул:

- `new ObjectPool(prefab, parent, prewarm)` — создаёт пул и прогревает `prewarm` экземпляров.
- `pool.Get(position, rotation)` — достать/создать.
- `pool.Release(instance)` — вернуть в пул (деактивирует и убирает под `parent`).

Хорошие кандидаты на пул: пули, искры, частицы, враги-респавнеры.

## Теги и слои

В `ProjectSettings` уже есть:

- **Tags:** `Player`, `Enemy`, `Pickup`, `Interactable`, `Projectile`.
- **Physics Layers:** `Player`, `Enemy`, `Projectile`, `Interactable`, `Ground` (+ Default).
- **Sorting Layers (2D):** `Background → Midground → Entities → Foreground → UI`.

Добавлять новое — через `Edit → Project Settings → Tags and Layers`.

## Структура игрового кода

Клади в `Assets/_Project/Scripts/Gameplay/`. Рекомендуемое разделение:

```
Gameplay/
  Player/       PlayerController · PlayerHealth
  Enemies/      EnemyAI · EnemySpawner
  Items/        Pickup · Interactable
  Systems/      Spawner · Scoring · Difficulty
```

Неймспейс — `LudumDare.Template.Gameplay` (или свой бренд игры, если сильно хочется — но тогда поменяй во всём проекте разом).

## Префабы

- Геймплей-префабы → `Assets/_Project/Prefabs/Gameplay/`.
- UI-префабы (если нужны отдельно от сцены) → `Assets/_Project/Prefabs/UI/Screens/`.
- Менеджеры — **не** клади новые префабы рядом с `_Managers.prefab`; просто добавляй компонент **внутрь** самого `_Managers.prefab`.

## Быстрые чек-листы

**Добавить звук при действии:**
1. `AudioCueSO` ассет (см. [`04_Звук_и_AudioCue.md`](04_Звук_и_AudioCue.md)).
2. `[SerializeField] AudioCueSO _cue;` в компоненте.
3. `AudioManager.Instance.PlaySFX(_cue);`.

**Добавить кастомное событие геймплея:**
1. Выбери тип канала (Int/Float/Void/String/Custom).
2. Создай `.asset` в `ScriptableObjects/EventChannels/`.
3. Издатель: `channel.Raise(...)`. Подписчик: `channel.OnEventRaised += Handler` в `OnEnable`, `-=` в `OnDisable`.
