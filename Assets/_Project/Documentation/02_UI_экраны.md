# 02. UI-экраны

## Как работает `UIScreen`

Базовый класс [`Assets/_Project/Scripts/UI/UIScreen.cs`](../Scripts/UI/UIScreen.cs) оборачивает `CanvasGroup`-фейд:

- В `Awake` экран выключается (`gameObject.SetActive(false)`, `alpha = 0`) — именно поэтому **сам по себе** ни один экран не видно.
- В `Start` экран **показывается автоматически**, если в инспекторе выставлены флаги:
  - `_showOnStart = true` — показать сразу при загрузке сцены.
  - `_replacesStack = true` — очистить стек `UIManager` и стать единственным видимым (иначе обычный `Push` поверх).
- Метод `Show()`/`Hide()` плавно меняет `alpha` и `interactable`/`blocksRaycasts`. Деактивирует GameObject в конце `Hide`.

По умолчанию `_showOnStart = false`. Включают его **только у стартового экрана сцены** — в шаблоне это `MainMenuScreen` (в `01_MainMenu.unity`) и `HUDScreen` (в `02_Game.unity`). Остальные (Settings, Pause, EndGame, Loading) показываются руками через `UIManager`.

## `UIManager`

[`Assets/_Project/Scripts/UI/UIManager.cs`](../Scripts/UI/UIManager.cs) — синглтон на `_Managers.prefab`. Держит:

- `_registered` — список всех экранов, зарегистрированных в текущей сцене (для `Find<T>()`).
- `_stack` — стек видимых экранов.

Ключевые методы:

- `Push(screen, hideCurrent = true)` — поверх, прячет текущий.
- `Pop()` — снимает верхний, показывает предыдущий.
- `Replace(screen)` — очищает весь стек и ставит один экран.
- `CloseAll()` — закрывает всё.

## Добавить новый экран (рецепт)

1. Создай класс, наследующий `UIScreen`:
   - Разместить в `Assets/_Project/Scripts/UI/MyScreen.cs`, неймспейс `LudumDare.Template.UI`.
   - В `Awake` вызови `base.Awake()`, повесь слушатели кнопок.
   - Для реакции на показ/скрытие используй `OnShow`/`OnHide`.
2. В сцене/префабе добавь GameObject с `CanvasGroup` + твоим компонентом, внутри — UI-контент.
3. Показ:
   - Одноразово из меню: `UIManager.Instance.Push(_myScreen)` с `[SerializeField] MyScreen _myScreen;`.
   - Как overlay (не прятать текущий): `UIManager.Instance.Push(_myScreen, hideCurrent: false)`.
   - Если это стартовый экран сцены — просто выстави `_showOnStart = true` в инспекторе.

## Поведение по сценам

| Сцена | Стартовый экран | Флаги |
|---|---|---|
| `01_MainMenu` | `MainMenuScreen` | `_showOnStart = true`, `_replacesStack = true` |
| `02_Game`     | `HUDScreen`      | `_showOnStart = true`, `_replacesStack = true` |

`SettingsScreen` живёт рядом с `MainMenuScreen` под тем же `UIRoot` и показывается через `UIManager.Push(_settingsScreen)` из `MainMenuScreen.OnSettings()`.

`PauseScreen` показывается через `PauseInputWatcher`, когда игрок нажимает ESC.

`EndGameScreen` подписан на `OnGameStateChanged` и сам `Push`-ится поверх HUD при `GameOver/Victory`.

## Часто встречающиеся грабли

- **Экран не видно.** Проверь: `CanvasGroup` есть, его не скрыл внешний `CanvasGroup`-родитель, `_showOnStart` включен у стартового экрана сцены или кто-то вызвал `Push/Replace`.
- **Кнопки не кликаются.** Во время fade-in `interactable = false`. Если кнопки не работают и после — проверь `blocksRaycasts`, `EventSystem` в сцене, и что ничего не перекрывает сверху (другой экран с `blocksRaycasts=true`).
- **Экран мигнул и пропал.** Скорее всего кто-то вызвал `Push` поверх него с `hideCurrent = true`.
