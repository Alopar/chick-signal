# 00. Старт

Краткий гид, что сделать после клона репозитория и как запустить игру.

## Требования

- Unity **6000.4.2f1** (или совместимая 6000.x).
- URP с **2D Renderer**, Input System, Cinemachine 3 — уже прописаны в `Packages/manifest.json`.

## Первый запуск

1. Открой проект в Unity Hub. Дождись импорта ассетов.
2. `Window → TextMeshPro → Import TMP Essential Resources`. «Examples & Extras» ставить не обязательно.
3. (Опционально) DOTween:
   - `Package Manager → + → Install package from git URL →` `https://github.com/Demigiant/dotween.git`, **или**
   - скачать `.unitypackage` с <https://dotween.demigiant.com/download.php> и импортировать в `Assets/Plugins/Demigiant/`.
   - После импорта: `Tools → Demigiant → DOTween Utility Panel → Setup DOTween…`.
   - Проект компилируется и без DOTween: UI-фейды работают на корутинах.

## AudioMixer (обязательный шаг)

Микшер `Assets/_Project/Audio/Mixers/MainMixer.mixer` уже создан. В нём три группы: `Master`, `Music`, `SFX`. Нужно выставить три **exposed-параметра** (это то, что читает скрипт `AudioManager`):

1. Открой `MainMixer.mixer` (окно Audio Mixer).
2. Выбери группу `Master` → в Inspector правой кнопкой по значению **Volume** → `Expose 'Volume (of Master)' to script` → переименуй exposed-параметр в **`Master`**.
3. Повтори для `Music` → **`Music`** и `SFX` → **`SFX`**.

Имена exposed-параметров должны точно совпадать с именами групп: `Master`, `Music`, `SFX`. Без этого ползунки в `Settings` не будут двигать громкость.

> Частая путаница: имя **группы** (что в дереве микшера) и имя **exposed-параметра** (что видит код) — это разные вещи. Мы сознательно сделали их одинаковыми, чтобы не запутаться.

## Запуск

1. Открой сцену `Assets/_Project/Scenes/00_Bootstrap.unity`.
2. Нажми Play.

Поток сцен:

```
00_Bootstrap → 01_MainMenu → 02_Game (с EndGameScreen как overlay)
```

Все сцены уже добавлены в **Build Settings** в правильном порядке.
