# 05. Сохранения и настройки

Для jam-масштаба всё держим на `PlayerPrefs` через обёртку [`SaveManager`](../Scripts/Managers/SaveManager.cs).

## Что сохраняется из коробки

| Свойство | Тип | Ключ PlayerPrefs | По умолчанию |
|---|---|---|---|
| `MasterVolume` | `float [0..1]` | `audio.master` | `1.0` |
| `MusicVolume` | `float [0..1]` | `audio.music` | `0.8` |
| `SfxVolume` | `float [0..1]` | `audio.sfx` | `1.0` |
| `Fullscreen` | `bool` | `video.fullscreen` | `true` |
| `BestScore` | `int` | `gameplay.bestScore` | `0` |

Все сеттеры вызывают `PlayerPrefs.Save()` автоматически.

`TrySetBestScore(int candidate)` обновляет рекорд, только если `candidate > BestScore`, и возвращает `true` если обновил.

## Добавить новое сохраняемое поле

1. В `SaveManager.cs` добавь приватную константу `KeyXxx` и публичное свойство с `PlayerPrefs.Get*`/`Set*`.
2. Если тип составной (json, коллекция) — сериализуй через `JsonUtility` в `PlayerPrefs.SetString`.

## SettingsScreen

[`SettingsScreen.cs`](../Scripts/UI/SettingsScreen.cs):

- В `OnShow` читает текущие значения из `SaveManager` и заливает в `Slider`/`Toggle` через `SetValueWithoutNotify` (чтобы не триггернуть `onValueChanged`).
- При изменении ползунков зовёт `AudioManager.Instance.Set*Volume` — а тот уже пишет в `SaveManager`.
- Toggle `Fullscreen` напрямую пишет в `SaveManager.Fullscreen` и меняет `Screen.fullScreen`.

## Очистить сохранения

Для отладки: `Edit → Clear All PlayerPrefs` в Unity, либо руками в коде: `PlayerPrefs.DeleteAll()`.
