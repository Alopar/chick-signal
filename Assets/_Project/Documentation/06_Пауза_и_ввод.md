# 06. Пауза и ввод

## PauseService

[`Assets/_Project/Scripts/Managers/PauseService.cs`](../Scripts/Managers/PauseService.cs) — синглтон на `_Managers.prefab`.

- `Pause()` — `Time.timeScale = 0` + `Raise(_onPaused)`.
- `Resume()` — `Time.timeScale = 1` + `Raise(_onResumed)`.
- `Toggle()` — переключает.
- `IsPaused` — текущее состояние.
- Автоматически восстанавливает `Time.timeScale` при уничтожении.

В инспекторе `_onPaused` и `_onResumed` ссылаются на ассеты `OnGamePaused.asset` / `OnGameResumed.asset`.

> Всегда паузься через `PauseService`. Прямая правка `Time.timeScale` ломает инвариант — другие системы подписаны на события.

## PauseInputWatcher

[`Assets/_Project/Scripts/UI/PauseInputWatcher.cs`](../Scripts/UI/PauseInputWatcher.cs) — живёт в сцене `02_Game.unity`. На `ESC` (клавиатура) или Start (геймпад):

- Если не в паузе — `PauseService.Pause()` + `UIManager.Push(_pauseScreen, hideCurrent: false)` (overlay поверх HUD).
- Если в паузе — `PauseService.Resume()` + `UIManager.Pop()`.

## InputReader

`Assets/_Project/Scripts/Input/` содержит SO-обёртку над Input System Actions. Для jam-геймплея это опциональная абстракция — многие пишут через `InputSystem_Actions.inputactions` напрямую.

Рекомендуемый паттерн:

- Одно действие (`Move`, `Jump`, `Interact`) — одно событие/свойство на `InputReader`.
- Геймплей-код читает `InputReader`, а не `Keyboard.current`. Это упрощает ребинды и тестирование.

## Использование `Time.timeScale`-независимого времени

Все UI-фейды и корутины в шаблоне используют `Time.unscaledDeltaTime` / `Time.unscaledTime`. То есть даже на паузе они продолжают работать. Если пишешь свою корутину, где нужно игнорировать паузу — тоже используй unscaled-таймеры.
